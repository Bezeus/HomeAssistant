# HomeAssistant
Сборка HomeAssistant в докер контейнер

## Запущенные сервисы и порты

Ниже представлен список всех контейнеров в сборке, их назначение и порты для доступа (если применимо).

| Сервис | Порт | Описание |
|---|---|---|
| **Home Assistant** | `8123` (host) | Главный хаб умного дома. Доступен по порту 8123 (использует network_mode: host). |
| **MariaDB** | `3306` | База данных для хранения истории состояний Home Assistant. |
| **phpMyAdmin** | `8090` | Веб-интерфейс для управления базой данных MariaDB. |
| **Mosquitto** | `1883` | MQTT-брокер для общения между устройствами (например, Zigbee2MQTT) и Home Assistant. |
| **Zigbee2MQTT** | `8020` | Веб-интерфейс сервиса управления Zigbee-сетью. |
| **Node-RED** | `1880` | Визуальная среда для создания сложных автоматизаций. |
| **ESPHome** | `6052` (host)* | Панель управления прошивками ESP-устройств (использует network_mode: host, дефолтный порт 6052). |
| **Code-Server** | `8443` | Веб-версия VS Code для удобного редактирования конфигурационных файлов прямо в браузере. |
| **Portainer** | `9000` / `9443` | Панель управления Docker-контейнерами (HTTP и HTTPS соответственно). |
| **Hass Configurator** | `3218` | Легкий веб-редактор файлов (File Editor) конфигурации Home Assistant. |
| **Matter Server** | `5580` (host)* | Сервер протокола Matter для работы с умными устройствами (использует network_mode: host, дефолтный порт 5580). |

*\* Сервисы с указанием (host) работают в режиме `network_mode: host` и используют дефолтные порты своих приложений на сетевом интерфейсе сервера.*
